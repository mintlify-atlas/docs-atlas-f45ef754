## Overview

The LK (Little Kernel) payload is injected into the device's LK bootloader to provide persistent exploit capabilities. Unlike Stage 1 and Stage 2 which run in BROM context, the LK payload executes during normal boot, allowing it to manipulate boot modes, patch security checks, and force fastboot access.

### Execution Context

- **Load Address**: `0x41E00000` (LK_BASE)
- **Size**: 1 MB (LK_SIZE)
- **Trigger**: Executed by microloader during every boot
- **Environment**: Early boot, before Android/Recovery loads
- **Persistence**: Written to Boot0 partition, survives reboots

## Purpose

The LK payload enables:

1. **Fastboot Access**: Force device into fastboot mode without user interaction
2. **Boot Mode Control**: Override boot mode based on MISC partition flags
3. **Security Bypass**: Patch device unlock and command verification
4. **Boot Image Demangling**: Fix malformed boot images (anti-downgrade protection)
5. **Persistent Backdoor**: Maintain exploit access after power cycles

## GPT Parsing

The payload scans the GPT to locate critical partitions:

**Implementation** (main.c:56-79):
```c
static void parse_gpt() {
    uint8_t raw[0x800] = { 0 };
    struct device_t *dev = get_device();
    dev->read(dev, 0x400, raw, sizeof(raw), USER_PART);
    
    for (size_t i = 0; i < sizeof(raw) / 0x80; ++i) {
        uint8_t *ptr = &raw[i * 0x80];
        uint8_t *name = ptr + 0x38;
        uint32_t start;
        memcpy(&start, ptr + 0x20, 4);
        
        if (memcmp(name, "b\x00o\x00o\x00t\x00\x00\x00", 10) == 0) {
            printf("found boot at 0x%08X\n", start);
            g_boot = start;
        } else if (memcmp(name, "r\x00\x65\x00\x63\x00o\x00v\x00\x65\x00r\x00y\x00\x00\x00", 18) == 0) {
            printf("found recovery at 0x%08X\n", start);
            g_recovery = start;
        } else if (memcmp(name, "l\x00k\x00\x00\x00", 6) == 0) {
            printf("found lk at 0x%08X\n", start);
            g_lk = start;
        } else if (memcmp(name, "M\x00I\x00S\x00\x43\x00\x00\x00", 10) == 0) {
            printf("found misc at 0x%08X\n", start);
            g_misc = start;
        }
    }
}
```

else if(g_misc) {

## Security Patches

### Device Unlock Bypass

Patch the unlock verification function to always return "unlocked":

**Implementation** (main.c:211-214):
```c
// device is unlocked
patch = (void*)0x41E01D34;
*patch++ = 0x2001; // movs r0, #1
*patch = 0x4770;   // bx lr
```

**ARM Thumb Assembly**:
```asm
0x41E01D34:  movs r0, #1    ; Return 1 (unlocked)
0x41E01D36:  bx lr          ; Return from function
```

### Fastboot Command Verification

Bypass Amazon's command restrictions in fastboot:

**Implementation** (main.c:216-219):
```c
// amzn_verify_unlock (enable all commands)
patch = (void*)0x41E01EA0;
*patch++ = 0x2000; // movs r0, #0
*patch = 0x4770;   // bx lr
```

**ARM Thumb Assembly**:
```asm
0x41E01EA0:  movs r0, #0    ; Return 0 (success)
0x41E01EA2:  bx lr          ; Return from function
```

This allows unrestricted fastboot commands:
- `fastboot oem unlock`
- `fastboot flash`
- `fastboot erase`
- `fastboot boot`

## Boot Image Demangling

Amazon devices use malformed boot images with offset headers to prevent downgrade. The payload hooks the read function to fix them:

### Read Hook Installation

**Implementation** (main.c:221-225):
```c
// hook bootimg read function
uint32_t *patch32;
original_read = (void*)dev->read;
patch32 = (void*)&dev->read;
*patch32 = (uint32_t)read_func;
```

### Read Function Hook

**Implementation** (main.c:32-54):
```c
size_t read_func(struct device_t *dev, uint64_t block_off, void *dst, 
                 uint32_t sz, uint32_t part) {
    printf("read_func hook\n");
    int ret = 0;
    
    if (block_off == g_boot * 0x200 || block_off == g_recovery * 0x200) {
        printf("demangle boot image - from 0x%08X\n", __builtin_return_address(0));

        if (block_off == g_boot * 0x200 && *y_cable == 1 
            && !boot_system && *boot_reason == 0) {
            printf("Boot into recovery to display bootmenu");
            block_off = g_recovery * 0x200;
        }
        
        if (sz < 0x400) {
            ret = original_read(dev, block_off + 0x400, dst, sz, part);
        } else {
            void *second_copy = (char*)dst + 0x400;
            ret = original_read(dev, block_off, dst, sz, part);
            memcpy(dst, second_copy, 0x400);  // Move header to correct position
            memset(second_copy, 0, 0x400);    // Clear duplicate
        }
    } else {
        ret = original_read(dev, block_off, dst, sz, part);
    }
    return ret;
}
```

**What it does**:
1. Intercepts reads to `boot` and `recovery` partitions
2. Boot image header is at offset 0x400 instead of 0x0
3. Copies header from 0x400 to 0x0 (where LK expects it)
4. Clears the duplicate header at 0x400

**Y-Cable Boot Menu**:
If a Y-cable is detected (`y_cable == 1`) and boot reason is 0, automatically boot recovery to show the boot menu.

## LK Memory Restoration

Restore the portion of LK overwritten by microloader:

**Implementation** (main.c:118-121):
```c
// Restore the 0x41E00200-0x41E01200 range, overwritten by microloader
dev->read(dev, g_lk * 0x200 + 0x200 + 0x200, 
          (char*)LK_BASE + 0x200, 0x1000, USER_PART);
```

The microloader overwrites `0x41E00200-0x41E01200` (4 KB) when loading the payload. This must be restored before jumping to LK.

<Warning>
Do not restore 0x41E00000-0x41E00200 as it contains important pointers set by preloader.
</Warning>

### Installing LK Payload

```python
# 1. Build LK payload
!cd lk-payload && make

# 2. Load via Stage 2

# Write LK payload at 0x80000
with open('lk-payload/build/payload.bin', 'rb') as f:
    payload = f.read()
    
for i, chunk in enumerate([payload[j:j+512] for j in range(0, len(payload), 512)]):
    block = 0x80000 // 512 + i
    stage2_write_block(block, chunk.ljust(512, b'\x00'))

```

## Persistence Model

The LK payload provides **persistent** exploit access:

```
[Power On] ---> BROM ---> Preloader ---> LK (with microloader)
                                          |
                                          v
                                    [Load LK Payload from Boot0]
                                          |
                                          v
                                    [LK Payload Executes]
                                          |
                                          v
                                    [Patch Security + Boot Mode]
                                          |
                                          v
                                    [Jump to Original LK]
                                          |
                                          v
                                    [Boot System/Recovery/Fastboot]
```

**Every boot**:
1. Microloader executes (injected into LK)
2. Microloader loads LK payload from Boot0
3. LK payload patches security and boot mode
4. LK payload jumps to original LK
5. Boot continues with patches active

This survives:
- Reboots
- Power cycles
- OTA updates (unless they overwrite Boot0)
- Factory resets
