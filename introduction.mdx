# Welcome to Kamakiri

Kamakiri is a powerful bootrom exploit tool designed for MediaTek devices, specifically targeting the Amazon Fire TV Stick 4K (Mantis). This tool enables bootloader unlock and custom recovery installation through low-level hardware exploitation.

<Warning>
  **Security Research Tool**: Kamakiri is designed for security research and device modification. Using this tool will void your warranty and may brick your device if used incorrectly. Proceed at your own risk.
</Warning>

## What is Kamakiri?

Kamakiri exploits a vulnerability in the MediaTek bootrom to gain low-level access to devices. Once access is obtained, the tool can:

- Bypass bootloader security checks
- Install custom recovery (TWRP)
- Unlock the bootloader without manufacturer authorization
- Enable fastboot mode without physical button combinations
- Modify partition tables and system partitions

## How It Works

Kamakiri works by exploiting the MediaTek bootrom's USB protocol handler. The exploit follows this sequence:

<Steps>
  <Step title="USB Connection">
    Connect to the device via USB serial interface while it's in bootrom mode
  </Step>
  <Step title="Bootrom Handshake">
    Perform the bootrom handshake protocol to establish communication
  </Step>
  <Step title="Payload Injection">
    Inject stage1 payload into bootrom memory using a buffer overflow vulnerability
  </Step>
  <Step title="Memory Access">
    Gain arbitrary memory read/write access through the stage1 payload
  </Step>
  <Step title="Stage2 Execution">
    Load and execute stage2 payload with full eMMC access capabilities
  </Step>
  <Step title="System Modification">
    Modify bootloader, recovery, and system partitions as needed
  </Step>
</Steps>

## Supported Devices

Currently, Kamakiri supports:

- **Amazon Fire TV Stick 4K (Mantis)** - Device type ID: `AKPGW064GI9HE`

The exploit targets MediaTek SoCs with specific bootrom vulnerabilities. Support for additional devices may be added in the future.

## Architecture Overview

Kamakiri consists of several components working together:

- **Python Modules**: High-level orchestration and USB communication (`modules/`)
- **Shell Scripts**: User-facing commands for common operations (`bootrom-step.sh`, etc.)
- **C Payloads**: Low-level code executed on the device (`brom-payload/`, `lk-payload/`, `microloader/`)
- **Binary Assets**: Precompiled bootloader and recovery images (`bin/`)
