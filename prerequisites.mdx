Before you begin using Kamakiri, ensure you have the necessary hardware, software, and system configurations in place.

## Hardware Requirements

<CardGroup cols={2}>
  <Card title="Compatible Device" icon="mobile">
    Kamakiri is designed for MediaTek MT8163 (Karnak) based devices, specifically targeting Amazon Fire tablets with device type ID `AKPGW064GI9HE` (Mantis).
  </Card>
  <Card title="USB Cable" icon="cable-car">
    A reliable USB data cable capable of both power delivery and data transmission. Quality matters for stable serial communication.
  </Card>
</CardGroup>

<Note>
  The exploit specifically checks for the device type ID `AKPGW064GI9HE` (Mantis). Other device models are not supported and will cause the exploit to exit with an error.
</Note>

## System Preparation

### Linux: Disable ModemManager

<Warning>
  **Critical**: ModemManager will interfere with the serial communication required for the bootrom exploit. You must disable or remove it before proceeding.
</Warning>

Kamakiri automatically checks for ModemManager at startup (`modules/common.py:24-34`):

```python
def check_modemmanager():
    pids = [pid for pid in os.listdir('/proc') if pid.isdigit()]
    for pid in pids:
        try:
            args = open(os.path.join('/proc', pid, 'cmdline'), 'rb').read().decode("utf-8").split('\0')
            if len(args) > 0 and "modemmanager" in args[0].lower():
                print("You need to temporarily disable/uninstall ModemManager before this script can proceed")
                sys.exit(1)
        except IOError:
            continue
```

To disable ModemManager:

<Tabs>
  <Tab title="Temporary (systemd)">
    ```bash
    sudo systemctl stop ModemManager
    sudo systemctl disable ModemManager
    ```
  </Tab>
  
  <Tab title="Permanent Removal">
    ```bash
    # Ubuntu/Debian
    sudo apt-get remove modemmanager
    
    # Fedora/RHEL
    sudo dnf remove ModemManager
    
    # Arch Linux
    sudo pacman -R modemmanager
    ```
  </Tab>
</Tabs>

Verify ModemManager is not running:

```bash
ps aux | grep -i modemmanager
```

### USB Permissions (Linux)

Add udev rules to allow non-root access to MediaTek devices:

```bash udev-rule.sh
echo 'SUBSYSTEM=="usb", ATTR{idVendor}=="0e8d", MODE="0666", GROUP="plugdev"' | sudo tee /etc/udev/rules.d/51-mediatek.rules
sudo udevadm control --reload-rules
sudo udevadm trigger
```

Add your user to the `plugdev` group:

```bash
sudo usermod -a -G plugdev $USER
```

<Note>
  You may need to log out and back in for group changes to take effect.
</Note>

### Serial Port Access

The exploit communicates over serial with these parameters:

- **Baud Rate**: 115200
- **Timeout**: 5 seconds
- **Vendor ID**: 0x0E8D (MediaTek)
- **Product ID**: 0x0003 (BootROM mode)

On Linux, verify you have access to serial ports:

```bash
sudo usermod -a -G dialout $USER
```
